KEY
  릴레이션에서 튜플들을 유일하게 구별하는 속성 및 집합
  조건에 만족하는 튜플 검색, 정렬, 구분의 기준이 되는 속성

1) 기본키 Primary Key
 - 후보키들 중 기본적으로 사용하기 위해 선택된 키
 - 하나의 릴레이션에서 특정 튜플을 구별할 수 있는 유일한 속성
 - (예) 학번

2) 후보키 Candidate Key
 - 기본키로 사용할 수 있는 속성들
 - 유일성과 최소성을 만족하는 속성 및 집합
 - (예) 학번, 주민번호

3) 대체키 Alternate Key
 - 후보키가 둘 이상일 때 기본키로 선택되지 못한 후보키(보조키)
 - (예) 주민번호

4) 슈퍼키 Super Key
 - 유일성을 만족하는 속성 및 집합
 - 최소성은 만족하지 않는다.
 - (예) 학번, 학번+이름, 학번+이름+주민번호

5) 외래키 Foreign Key
 - 다른 릴레이션의 기본키를 참조하는 속성 및 집합
 - 참조되는 릴레이션의 기본키와 대응되어 릴레이션 간 참조 관계를 표현하는 도구

제약 조건
1) 포함되어 있는 Implicit constraints
 - 릴레이션 자체적으로 가지고 있는 제약 조건
 - 릴레이션은 중복된 튜플을  가질 수 없다.
 - 릴레이션은 같은 이름의 속성을 가질 수 없다.

2) 무결성 제약 조건 Integrity 
 - 무결성 : 데이터의 결함이 없는 상태,  데이터의 정확성
 - 도메인 무결성 제약 조건  : 어트리뷰트의 값은 정의된 도메인에 속한 값이어야 한다.
 - 개체 무결성 제약 조건 : 기본키 제약 조건, 기본키를 구성하는 모든 속성은 NULL값과 중복값을 가질 수 없다.
 - 참조 무결성 제약 조건 : 외래키는 참조하고 있는 테이블의 기본키에 없는 값을 가질 수 없다.
 - Key Constraint : 서로 다른 튜플들은 값은 값의 KEY를 가실 수 없다.
 - NULL Value Constraint : 어트리뷰트가 NOT NULL이라면, NULL값을 가질 수 없다.

JOIN
테이블이 기본키와 외래키로 연결되어 있는 상태

INNER JOIN
 - 가장 많이 사용
 - 두 테이블이 공통적으로 가지고 있는 데이터 조회
예)SELECT
	검색할_열
FROM
	테이블1
	INNER JOIN
	테이블2
	ON
	조건 (기본키 = 외래키)
UNION
데이터 조회 결과 합치는 것. (위에서 아래로)
SELECT 하는 컬럼 개수가 동일할 것.
컬럼의 이름이 동일할 것. (동일하지 않으면 AS 사용)
컬럼의 데이터 값이 동일할 것.
MySQL 에서 규칙이 무시됨.

UNION
 - 중복 데이터 제외 출력

UNION ALL
 - 중복 데이터 포함 출력
예)SELECT
	컬럼1
	컬럼2
FROM
	테이블1
UNION
SELECT
	컬럼3
	컬럼4
FROM
	테이블2

NULL값 처리﻿
NULL값을 찾을 때 IS NULL을 사용
NULL아닌 값 찾을 때 IS NOT NULL 사용
예)컬럼 IS NULL;
컬럼2 IS NOT NULL;

IFNULL
- SELECT 절에서 NULL 데이터 조회 및 다른 값으로 반환
예)SELECT
	열1,
	IFNULL(user,'UNKNOWN')
FROM
	테이블1


릴레이션 스키마 변환 규칙
 - 모든 개체는 릴레이션으로 변환
 - 다대다 관계는 릴레이션으로 변환 
 - 일대다 관계는 외래키로 표현
 - 일대일 관계는 외래키로 표현
 - 다중 값 속성은 릴레이션으로 변환
